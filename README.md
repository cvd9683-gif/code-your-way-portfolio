# Code Your Way Portfolio

A 14-week structured build of my personal portfolio website — created to bridge the gap between product design and front-end development.

**Current Status: Week 5 of 14**

---

## 🧭 Project Overview

Code Your Way is not just a portfolio project. It is a deliberate learning experience designed to deepen my understanding of how interactive web experiences are built.

This site exists to help me:

- Move beyond no-code tools
- Understand HTML, CSS, and JavaScript at a foundational level
- Design interactions intentionally
- Build fluency in collaborating with developers
- Create a professional portfolio that reflects product thinking

By Week 14, this will be a live, deployable, interactive portfolio.

## 🎯 Why This Project Exists

Previously, I relied heavily on website builders (like Framer) to create interactive portfolios. While powerful, they limited my understanding of how interactions actually functioned.

I often had to choose between:
- How something looked
- How it behaved

This project changes that.

The goal is to become a designer who:
- Understands how things are built
- Can think in systems
- Can collaborate confidently with engineers
- Can build small interactive components independently

This portfolio is both a professional artifact and a technical learning record.

## 🧠 Design Values

This portfolio is guided by five core principles:

### 1. Connection  
Design begins with understanding users.

### 2. Interaction  
Motion and feedback are central to experience.

### 3. Clarity with Personality  
Clean and understandable, but not sterile.

### 4. Collaboration  
Great work is shaped through dialogue.

### 5. Trust  
Open to iteration without losing direction.

---

## ✨ Experience Principles

- Scrolling should feel like entering a responsive space.
- Motion should support clarity.
- Interactions should feel intentional.
- Animation should never compete with content.
- The site should feel alive, not chaotic.

---

## 🚫 What This Portfolio Is NOT

- Not a template-based site
- Not a static case-study dump
- Not an art experiment disconnected from UX
- Not over-engineered
- Not feature-heavy without purpose

If it doesn’t support product thinking, it doesn’t ship.

## 🛠 Technical Stack

Primary technologies:

Optional (limited use only if necessary):
- React

Out of scope:
- Backend systems
- Databases
- Heavy frameworks
- Over-engineered animation libraries

Workflow:

Figma → HTML → CSS → JavaScript → Deploy

Design decisions always come before implementation.
